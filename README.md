# 🌿 MediCanna AI: Medical Cannabis Efficacy & Side Effects Clustering Recommender

## 🚀 快速開始 (Quick Start)

1. **Bootstrap ML artifacts**（若缺少模型檔自動訓練並驗證）  
   `cd ml-python-engine && python -m venv venv && source venv/bin/activate && pip install -r requirements.txt && python bootstrap_models.py`

2. **本地開發（環境化設定）**  
   - ML API: `cd ml-python-engine && ML_FAIL_FAST=true uvicorn main:app --reload --port 8000`  
   - Gateway: `cd backend-rust-gateway && ML_PREDICT_URL=http://localhost:8000/api/predict ALLOWED_ORIGINS=http://localhost:4200 cargo run`（Port 8080）  
   - 前端: `cd frontend-angular && npm start`（Port 4200）  
   - 瀏覽 http://localhost:4200 輸入症狀取得推薦。

3. **Docker 一鍵啟動（含 healthcheck + readiness）**  
   `docker-compose up --build`  
   - 前端: http://localhost:80  
   - Gateway: http://localhost:8080  
   - ML: http://localhost:8000  

4. **驗證 API**  
   `bash scripts/verify-api.sh`（需先啟動服務）

5. **測試與 Contract Check**  
   - Rust: `cd backend-rust-gateway && cargo test`  
   - ML: `cd ml-python-engine && pytest -q`  
   - Frontend: `cd frontend-angular && npm test -- --watch=false --browsers=ChromeHeadless`  
   - Contract: `bash scripts/contract-test.sh`

## 📝 專案深度解析 / Comprehensive Project Overview

本專案致力於打造一個具備高度擴展性與實用價值的醫療大麻與管制藥品推薦系統，這不僅僅是一個學術展示，更是一個能夠實際解決病患痛點的商業級微服務架構應用。在現代醫療體系中，面對種類繁多、化學成分各異的合法醫療大麻品種，許多病患往往只能依靠試錯的方式來尋找最適合自己症狀的藥物，這不僅耗費金錢，更可能帶來不必要的副作用與身體負擔。為了解決這個資訊不對稱的難題，我們結合了當今最先進的機器學習技術與自然語言處理（NLP），並透過強大的前後端分離架構，將這個複雜的數據運算過程轉化為使用者友善的直覺介面。使用者只需要輸入他們目前的具體症狀、希望達到的療效，以及希望避免的副作用，系統就能在毫秒之間，從成千上萬的歷史數據與科學化驗結果中，精準篩選並推薦出最合適的品種。這套系統的誕生，象徵著資料科學在精準醫療領域的具體實踐，並且完美契合了 Glasgow 軟體開發碩士課程對於系統工程、數據分析與人工智慧核心能力的綜合要求，是一個充滿野心且極具潛力的頂級作品。

全球醫療大麻市場正處於前所未有的爆發性成長階段，隨著英國、加拿大以及美國多個州陸續將其合法化並納入正規醫療處方體系，這個產業的估值早已突破百億美元的規模。然而，市場的快速擴張也伴隨著嚴重的資訊混亂與產品標準化問題。傳統的醫療大麻分類往往只粗略分為 Sativa、Indica 或 Hybrid，但現代藥理學研究已經證實，真正決定療效與副作用的關鍵在於 THC 與 CBD 的精確比例，以及高達上百種的萜烯（Terpenes）成分組合。這意味著傳統的分類方式已經無法滿足精準醫療的需求。本專案正是瞄準了這個龐大的市場缺口與商業痛點，透過收集並分析公開資料庫中的海量使用者反饋與實驗室化驗數據，我們能夠挖掘出隱藏在龐雜資訊背後的關聯性。這不僅對於終端消費者（病患）有著巨大的價值，對於醫療提供者、合法藥局，甚至是藥物研發機構來說，這套系統所產出的數據洞察與分群結果，都能作為重要的決策依據，進而推動整個合法醫療大麻產業朝向更科學、更透明、更以病患為中心的未來發展方向。

在開發任何涉及醫療健康與管制藥品的軟體系統時，法規遵循（Regulatory Compliance）與數據隱私（Data Privacy）絕對是凌駕於所有技術考量之上的最高指導原則。本專案在架構設計之初，便將這些關鍵因素納入核心考量，確保系統在合法合規的前提下穩定運作。首先，我們處理的數據雖然來自公開的 Kaggle 資料集，但在模擬真實商業情境時，我們必須假設系統會收集到病患極度敏感的個人健康資訊（PHI），包含詳細的病史、用藥紀錄與生理反應。因此，在 Rust API Gateway 的設計中，我們必須實作最嚴格的身份驗證與授權機制（例如 OAuth 2.0 與 JWT 雙重驗證），並確保所有資料在傳輸過程（TLS/SSL）與靜態儲存時都經過高強度的加密處理。此外，針對醫療大麻的特殊性，系統的推薦邏輯必須明確標示為「資訊輔助參考」，而非「絕對的醫療診斷」，以避免觸犯各國嚴格的醫療法規紅線。透過在架構層面落實這些資安與合規措施，我們不僅保護了使用者的隱私，也為這個專案奠定了能夠真正邁向商業化運轉的堅實基礎，展現了專業軟體工程師應有的嚴謹態度。

整個推薦系統的智慧大腦建立在先進的自然語言處理（NLP）與非監督式學習（Unsupervised Learning）演算法之上。我們深知，單純依靠實驗室提供的化學成分數據來進行推薦是不夠的，因為真實世界的療效往往隱藏在成千上萬條使用者的文字評論之中。因此，我們首先利用 NLP 技術對這些非結構化的文本數據進行深度剖析。透過分詞、詞性標註、去除停用詞等預處理步驟，我們將雜亂無章的評論轉化為乾淨的特徵向量。接著，我們運用 TF-IDF 或更先進的詞嵌入（Word Embeddings）技術，提取出關於「療效」（如疼痛緩解、肌肉放鬆）與「副作用」（如口乾、焦慮、頭暈）的關鍵字頻率。在取得這些量化特徵後，我們引入 K-Means 分群演算法，讓 AI 根據化學成分與文本特徵的相似度，自動將數百種大麻品種劃分為多個具備獨特屬性的群集。這種資料驅動的分群方式，能夠打破傳統人工分類的盲點，發現那些化學成分相近且使用者反饋高度一致的「隱藏藥物家族」，進而大幅提升推薦系統的精準度與可靠性。

在機器學習與自然語言處理模組的開發上，我們毫不猶豫地選擇了 Python 作為核心技術堆疊。這並非盲目跟從潮流，而是基於深思熟慮的軟體工程考量與學術對標需求。首先，Python 擁有全球最龐大且活躍的資料科學開源生態系，我們可以直接調用 Pandas 進行高效的數據清洗與轉換，利用 Scikit-learn 快速實作與調校 K-Means 分群模型，並借助 NLTK 或 SpaCy 來處理複雜的英文文本分析任務。如果我們堅持使用其他語言從頭打造這些底層工具，將會耗費大量且不必要的時間成本。更重要的是，選擇 Python 完美契合了 Glasgow 軟體開發碩士課程對於機器學習核心技術的要求，讓您在入學前就能透過這個專案徹底熟悉該課程的開發工具與思維模式，將理論知識轉化為實戰經驗。為了與其他微服務無縫接軌，我們將這個 Python 模組封裝為 FastAPI 微服務，利用其異步處理能力與自動生成 API 文件的特性，打造出一個高效能、易於維護且擴展性極強的機器學習推論引擎。

將 Rust 引入本專案作為後端 API Gateway，是整個微服務架構中最具前瞻性與技術挑戰性的決策，這將使您的作品在眾多學生的專案中脫穎而出。Rust 以其無可匹敵的記憶體安全性（Memory Safety）與極致的執行效能聞名於世，完美解決了傳統系統編程語言容易發生的記憶體外洩或數據競爭等致命問題。在這個系統中，Rust 將扮演著「強力中樞」的角色，負責承受來自前端 Angular 應用的所有併發請求。透過 Axum 或 Actix-Web 等高效能異步框架，Rust 能夠以極低的資源消耗處理海量連線，並負責繁重的身份驗證、請求頻率限制（Rate Limiting）與日誌記錄等基礎設施工作。當確認請求合法且安全後，Rust 會透過內部網路將資料快速轉發給後端的 Python AI 服務進行深度運算，並在取得結果後迅速回傳給前端。這種「Rust 負責高效路由與安全，Python 負責智能運算」的黃金組合，正是當前矽谷一線科技大廠用來處理高併發與大數據流的標準微服務架構模式，展現了極高的系統工程水準。

前端使用者介面的開發，我們選擇了由 Google 維護的企業級框架 Angular。面對這個資料結構相對複雜、需要呈現多種過濾條件與詳細藥品資訊的推薦系統，Angular 所提供的全面性解決方案與強型別（TypeScript）特性顯得尤為重要。有別於其他輕量級框架，Angular 內建了強大的依賴注入（Dependency Injection）、完整的路由機制（Routing）以及強大的表單處理模組（Reactive Forms），這讓我們能夠以高度模組化且具備良好測試性的方式來構建前端應用程式。透過 TypeScript 的嚴格型別檢查，我們可以在編譯階段就捕捉到潛在的資料傳遞錯誤，確保從 Rust 後端接收到的複雜 JSON 資料結構能夠被正確無誤地解析與渲染。此外，Angular 豐富的生態系中包含了許多高質量的 UI 元件庫（如 Angular Material），幫助我們快速打造出專業、美觀且具備響應式設計（Responsive Design）的醫療級應用介面，讓病患能夠在各種裝置上獲得一致且流暢的操作體驗。

宏觀來看，本專案的系統架構採用了經典的三層式微服務設計，旨在實現高內聚、低耦合的軟體工程理想狀態。這不僅大幅提升了系統的穩定性與可維護性，更為未來的水平擴展（Horizontal Scaling）留下了充足的空間。最外層是用戶直接互動的 Angular 前端應用（Client Layer），負責收集使用者的症狀輸入與渲染推薦結果；中間層是核心的 Rust API Gateway（API Layer），扮演著系統守門員的角色，處理所有業務邏輯、資料庫存取（若有記錄使用者歷史的需求）以及微服務間的通訊調度；最內層則是專注於數據運算的 Python 機器學習服務（Inference Layer），它只對內網開放，專心執行 NLP 分析與 K-Means 模型推論。這三個獨立的服務將透過 Docker 容器化技術進行打包與隔離，並可藉由 Docker Compose 實現一鍵部署。這種架構設計確保了當某個服務需要更新或擴容時（例如 AI 模型需要重新訓練），完全不會影響到其他服務的正常運行，體現了成熟軟體架構的強大韌性。

在人工智慧領域有一句名言：「垃圾進，垃圾出（Garbage In, Garbage Out）」，這凸顯了數據預處理與特徵工程在整個專案中的決定性地位。本系統的數據管道（Data Pipeline）設計極為嚴謹，旨在從 Kaggle 提供的原始 CSV 資料集中萃取出最具價值的黃金特徵。首先，我們會使用 Pandas 對資料進行基礎清洗，處理缺失值（Missing Values）與異常數據（Outliers）。接著，針對化學成分資料（如 THC/CBD 比例），我們會進行標準化（Standardization）或正規化（Normalization）處理，確保不同量級的數值在後續的距離運算中擁有公平的權重。而在文本處理方面，我們除了基礎的清理與詞幹提取外，還會建立專屬的「醫療大麻領域詞典」，以確保模型能夠正確理解特定術語（例如將不同拼法的同一種萜烯統一）。最終，這些經過精心打磨的化學數值特徵與文本向量特徵將被無縫合併，形成一個高維度的綜合特徵矩陣，為接下來的機器學習模型提供最高品質的運算燃料，確保分群結果的精準性。

深入探討自然語言處理模組的核心運作機制，我們面對的最大挑戰在於如何將充滿主觀情緒與模糊描述的使用者評論，轉化為客觀且可量化的醫療參考指標。為了達成這個目標，我們實作了一套多層次的文本分析流程。首先，我們利用 SpaCy 這個工業級的 NLP 函式庫進行詞性標註（POS Tagging）與命名實體識別（NER），精準抓取評論中與「症狀」、「療效」及「副作用」高度相關的詞彙。接著，我們採用了 TF-IDF（詞頻-逆向文件頻率）演算法來建立詞彙矩陣。有別於簡單的詞頻統計，TF-IDF 能夠自動降低那些常見但無意義的字彙的權重，並顯著提升那些罕見但關鍵的特定症狀描述（如失眠、偏頭痛、被害妄想）的影響力。為了解決同義詞與語意相近的問題，我們甚至可以在未來進階實作中引入更先進的預訓練模型，將詞彙映射到連續的語意空間中，讓 AI 不僅能「看見」文字，更能深入「理解」隱藏在文字背後的真實病患感受，極大地提升系統的智能化與同理心。

在將所有數據轉化為高維度特徵矩陣後，我們迎來了本專案的核心演算法：K-Means 分群（Clustering）。作為一種經典的非監督式學習演算法，K-Means 的任務是在沒有預先標註答案的情況下，自動找出資料中的潛在結構與群體。我們會將包含化學成分與 NLP 特徵的綜合矩陣輸入給演算法，並透過「手肘法（Elbow Method）」等科學評估指標，來決定最理想的群集數量。在運算過程中，演算法會不斷迭代調整每個群集的中心點，直到將數百種大麻品種完美劃分到各個最具代表性的群體中。這個分群結果將產生極為震撼的商業價值：系統會發現某個群集的特徵是高 CBD 且文本評論高度集中在「緩解焦慮」；而另一個群集則可能是針對「重度失眠」的特效組合。這讓我們的推薦不再是瞎子摸象，而是基於龐大數據支撐的精準科學分類，徹底顛覆傳統的藥物選擇模式。

將視角轉向 Rust API Gateway 的實作細節，這是一個追求極致效能與堅若磐石安全性的工程傑作。我們將採用目前 Rust 生態系中最受歡迎且效能頂尖的 Axum 框架來建構 HTTP 伺服器。在路由設計上，我們會嚴格遵循 RESTful API 的最佳實踐，定義清晰明確的端點，例如負責接收前端查詢條件的路由，以及負責提供特定藥物詳細資訊的介面。為了確保系統在高負載下的穩定性，我們會充分利用 Tokio 這個強大的異步運行時，讓 Rust 能夠在等待 Python 微服務回傳推論結果的空檔，繼續處理其他成千上萬的併發連線，達到非阻塞（Non-blocking）的完美境界。此外，我們將嚴格定義所有進出 API 的資料結構，並結合 Serde 套件進行高效的 JSON 序列化與反序列化。任何格式不符的惡意請求都會在到達業務邏輯層之前被 Rust 的型別系統無情阻擋，這正是使用 Rust 打造企業級後端微服務的最迷人之處與最大優勢。

在 Angular 前端介面的刻劃上，我們將秉持著「以人為本」的設計哲學，力求在複雜的醫療數據與直覺的用戶體驗之間取得完美平衡。首頁將被設計為一個極簡而優雅的問卷式引導介面，利用 Angular 強大的表單技術，捕捉使用者當下最迫切的緩解需求、可接受的副作用程度以及偏好的給藥方式。當用戶送出表單後，這些參數將透過 Angular 內建的 HTTP 模組以異步方式發送至 Rust 後端。在等待結果的極短時間內，我們會加入平滑的加載動畫，避免使用者的焦慮等待。當推薦結果回傳後，介面將透過卡片式的 UI 組件，清晰呈現出最匹配的精選品種推薦，並利用互動式圖表將該品種的化學成分比例與分群特徵以視覺化的方式展現出來。這種數據視覺化的設計，能夠讓毫無醫學背景的病患也能一眼看懂 AI 的推薦邏輯與數據佐證，建立起對這套醫療系統的強大信任感與安全感。

一個優秀的軟體專案，其價值必須透過成功的部署（Deployment）才能真正展現，而這也是展現您系統維運能力的絕佳舞台。為了確保開發環境與生產環境的絕對一致性，我們將全面導入 Docker 容器化技術。我們會為 Angular 前端、Rust 伺服器以及 Python AI 服務分別撰寫專屬的設定檔，精準控制每個微服務的運行環境與系統依賴。接著，我們會編寫一個統籌全局的微服務編排檔案，讓您只需輸入一行簡單的指令，就能在任何伺服器上瞬間啟動這套複雜的三層式架構。為了進一步模擬真實的軟體工程生命週期，我們甚至可以在版本控制系統上設定簡單的自動化工作流程。每當您推送新的程式碼到程式庫時，系統會自動執行程式碼檢查與容器映像檔的構建，確保每一次的版本更新都具備絕對的可靠性。這種將自動化思維融入開發流程的實踐，將會讓您的專案在未來的求職面試或學術評審中展現出高度的專業性。

站在未來的發展願景上，這個醫療大麻與管制藥品推薦系統具備著無限的擴展潛力與功能迭代空間。在現有的基礎之上，我們未來的開發藍圖可以朝著幾個極具挑戰性的方向邁進。首先是引入更複雜的推薦機制，當系統正式上線並累積足夠的使用者行為數據後，我們能將純粹基於成分與分群的推薦，升級為更智能的混合式推薦引擎，達到「像您這樣的病患，也覺得這款藥物有效」的高度個人化體驗。其次，我們可以擴展 Rust 後端的功能，整合外部的即時藥局庫存系統，讓病患不僅知道該用什麼藥，還能立刻知道哪裡可以合法取得。最後，針對機器學習模型本身，我們可以實作自動化的模型監控與再訓練機制，當市場上出現新的大麻品種或使用者反饋發生明顯改變時，系統能夠自動更新知識庫，確保推薦邏輯永遠保持在最先進的狀態。這套系統的未來，就是一個不斷自我進化、持續為病患創造龐大價值的智慧醫療生態系。
